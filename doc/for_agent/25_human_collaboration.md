# 25 — 인간 사용자 협업 프로토콜

> **관련:** `28_human_action_items.md` · `for_human/28_human_action_items.md` · `23_agent_operating_rules.md`  
> **원본:** `doit.md` §인간 사용자 협업 원칙 · **모든 에이전트 필독**

## 전제

AI는 모든 작업을 독립적으로 해결할 수 있다고 **가정하지 않는다**.  
불확실성이 크면 **임의 구현보다 명확한 요청**을 우선한다.

## 블로커 발생 시 — 미루고 계속 (필수)

아래 **12가지**에 해당하는 일이 생기면 **전체 작업을 멈추지 않는다**.

| 해야 할 일 | 하지 말아야 할 일 |
|------------|-------------------|
| `for_human/28_human_action_items.md` **미완료** 표에 항목 추가 | API 키·승인 없이 해당 기능을 “완료” 처리 |
| 블로커 TODO는 **미루고** 다른 AI 가능 작업 진행 | 가짜 키·무승인 RLS/프로덕션 배포 |
| (선택) 세션 말미 채팅에 **높음** 우선순위 미완료 1~2줄 요약 | 28번 문서에만 쌓아 두고 영원히 말하지 않기 |

**적용:**

- 계획·구현·테스트 중 인간 전용 단계를 만나면 → **28번에 기록** → **다음 AI 작업**으로 넘어감
- 「바로 해줘」도 **인간 전용** 작업(키, 결제, 법무, 프로덕션 GO)은 AI가 대신 완료할 수 없음

**정본 문서:** `doc/for_human/28_human_action_items.md`  
**에이전트 절차:** `doc/for_agent/28_human_action_items.md`

## 반드시 인간에게 요청해야 하는 상황 (12가지)

| # | 상황 | 예시 |
|---|------|------|
| 1 | 외부 서비스 가입 | Supabase, Firebase, Vercel, OpenAI 계정 |
| 2 | API Key 발급 | OpenAI, Supabase service role |
| 3 | 결제·비용 | 유료 플랜, 도메인 구매 |
| 4 | 법적 판단 | 개인정보, 이용약관 |
| 5 | 사업적 의사결정 | 타깃, 수익 모델 |
| 6 | 디자인 취향 | 로고, 색, 톤앤매너 최종 |
| 7 | 배포 승인·서비스 공개 | 프로덕션 오픈 GO/NO-GO |
| 8 | 계정·보안 인증 | 학교 SSO, 2FA 정책 |
| 9 | 물리적 장치 접근 | 실기기, 학교 서버실 |
| 10 | AI 권한 밖 작업 | 결제 실행, 법률 서명 |
| 11 | 방향성에 큰 영향 | Auth 통합 방식 전환 등 |
| 12 | 정보 부족·오구현 위험 | 요구사항 모호, 스키마 미정 |

→ 발생 시 **28번 미완료 표**에 한 줄 요약 + (필요 시) 아래 채팅 형식으로 안내.

**28에 넣지 않음 (AI가 MCP로 처리):** `supabase/migrations` DDL, Storage 버킷·RLS 초안, 컬럼 누락, `apply_remote_full` 원격 Run 요청. → [`33_supabase_mcp_db_operations.md`](./33_supabase_mcp_db_operations.md)

## 대화창 출력 형식 (선택 — 높음 우선순위·세션 말미)

긴급하거나 사용자가 물었을 때, 또는 세션 종료 요약 시:

```
[인간 협업 — 대기 목록]

28_human_action_items.md 에 추가했습니다.

미완료 (높음):
- H-00x: (한 줄)

다른 작업은 계속 진행했습니다: (한 줄 요약)

완료하시면: “H-00x 완료” 또는 키/승인 반영 후 알려주세요.
```

상세 단계가 필요하면 기존 형식 유지:

```
사용자가 해야 하는 작업:
1. ...
   - 클릭 위치: ...
   - 명령어: ...
   - 입력값: ...
   - 예상 결과: ...
```

## 인간이 최소 노력으로 할 수 있게 (doit.md)

28번 항목에 가능한 한 포함:

- **단계별 가이드** (1, 2, 3…)
- **클릭 위치** · **명령어** · **입력값** · **예상 결과**
- **완료 확인 방법** (표의 열)

## 금지

- 질문해야 할 상황을 숨기기
- 28번에 기록하지 않고 잊기
- 억지 추론으로 프로덕션 설정·결제·법무·키 발급 진행
- “아마 될 것”으로 RLS·배포 확정

## 완료 확인

인간이 “완료” / “H-00x 완료”라고 하면:

1. `28_human_action_items.md` — **완료한 일**로 이동, `완료 시각`
2. `.env` 등 반영 여부 확인 (커밋 제외)
3. `05_todo.md` · `17_handoff.md` 갱신
4. `25_ai_work_log.md` 맨 위에 기록 (`YYYY-MM-DD HH:mm:ss`)
