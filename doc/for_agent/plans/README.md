# 세션 업무 계획 (plans/)

> **관련:** `doc/starter.txt` §0 · `23_agent_operating_rules.md`

## 목적

업무 계획을 **채팅이 아니라 마크다운 파일**로 제출합니다.  
에디터 왼쪽에 열어 두면 Plan 모드처럼 계획이 대화 위로 밀리지 않습니다.

**대화창 직접 요청**(`vision.md`가 아닌 사용자 메시지): `YYMMDD-N.md` + **같은 턴**에 `29` §대화창 맨 위 `C-YYMMDD-N` · `25` 맨 위.  
→ 정본: [`chat_request_recording.md`](../chat_request_recording.md) (응답 전 §6 자가 점검).

## 파일 명명 (필수)

| 규칙 | 설명 |
|------|------|
| 형식 | `YYMMDD-N.md` (예: `260519-1.md`, `260519-2.md`, `260520-1.md`) |
| `YYMMDD` | 계획 **작성일** (로컬 날짜) |
| `N` | 그날 만든 계획의 **순번** (1부터 증가) |
| **금지** | 기존 계획 파일 **덮어쓰기** — 항상 새 파일 추가 |

각 계획 파일 **맨 위**에 다음을 반드시 기록한다.

```markdown
> **계획 생성 시각:** YYYY-MM-DD HH:mm:ss
```

(로컬 시각, 시·분·초까지. 예: `2026-05-19 23:42:59`)

## 파일 역할

| 파일 | 용도 |
|------|------|
| **`YYMMDD-N.md`** | **계획 정본** — 세션마다 새 파일 1개 |
| **`current_session_plan.md`** | **인덱스만** — 최신·전체 계획 목록 (본문 덮어쓰기 금지) |
| `_TEMPLATE.md` | AI가 복사해 채울 양식 |
| `archive/` | (선택) 장기 보관 |

## 인간 사용자

1. AI가 계획을 쓰면 채팅에 **새 파일 경로**와 한 줄 요약만 온다.
2. `current_session_plan.md` 를 열면 **최신 계획 파일 링크**가 보인다.
3. 해당 `YYMMDD-N.md` 를 연어 본문을 확인한다.
4. **「진행해」** / **「바로 해줘」** 로 착수 승인.

## AI 에이전트

1. 오늘 날짜로 `plans/` 안 기존 `YYMMDD-*.md` 개수를 확인해 다음 `N` 결정.
2. `_TEMPLATE.md` 구조로 **`YYMMDD-N.md` 새 파일** 작성 (상태: `draft`).
3. 파일 상단에 **계획 생성 시각** (시분초) 기록.
4. `current_session_plan.md` 인덱스 표만 갱신 (기존 계획 파일 수정·삭제 금지).
5. 채팅에는 3~5줄 요약 + **새 파일 경로** + 승인 요청만.
6. 종료 시 **해당 계획 파일** 하단에 실행 결과 작성, 상태 `done`; 인덱스 상태 갱신.

## Cursor Plan 모드와의 관계

Cursor **Plan 모드 UI**를 코드로 강제할 수는 없습니다.  
대신 **같은 워크스페이스 마크다운 파일**을 왼쪽 탭에 열어 두면 UX가 유사합니다.
