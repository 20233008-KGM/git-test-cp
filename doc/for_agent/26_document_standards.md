# 26 — 문서 작성 표준

> **원본:** `doit.md` §문서 작성 원칙 · §for_agent 규칙 · §유지보수

## 모든 문서가 만족해야 할 기준 (12항)

- [ ] 현대적·최신 개발 방식 반영
- [ ] 유지보수성 높음 (짧은 문단, 표, 링크)
- [ ] 확장성 (역할·기능 추가 시 섹션 추가 가능)
- [ ] 역할·책임 명확
- [ ] 프로젝트 구조 빠른 이해
- [ ] 신규 온보딩 용이
- [ ] 인수인계 용이 (`17`, `starter.txt`)
- [ ] 장기 운영 적합 (날짜·버전 표기)
- [ ] AI가 읽기 쉬움 (스캔 가능, H2/H3)
- [ ] 검색 친화적 (키워드, 일관된 파일명)
- [ ] 문서 간 참조 명확 (`> 관련:` 블록)
- [ ] **현재 상태** vs **할 일** 분리 (`02` vs `05`)

## 체크리스트 표기 (프로젝트 규칙)

| 표기 | 의미 |
|------|------|
| `[o]` | 완료 |
| `[ ]` | 미완료 |

`[x]` 는 사용하지 않음 (취소·오류와 혼동 방지).

`05_todo.md` 표 **상태** 열은 `done` 문자열 대신 **`[o]`** / **`[ ]`** / `진행 중` 만 사용.

## for_agent 문서 추가 규칙

- 빠른 스캔 (표·불릿 우선)
- 계층 구조 (H1 → H2 → H3)
- 작업 단위 명확 (TODO ID)
- 문맥 없이 이해 가능 (약어는 glossary 링크)
- 다음 액션 한 줄 이상

## 대화창에서 직접 요청한 작업

> **에이전트 정본 (필독):** [`chat_request_recording.md`](./chat_request_recording.md)

`vision.md`가 아닌 **Cursor 대화창** 요청 = 계획 · 실행 · **같은 턴**에 `29`+`25`+`plans` 기록.

### ID (`C-YYMMDD-N`)

- `29` §대화창 표에서 **오늘 날짜의 최대 N**을 보고 **N+1** 부여.
- vision `#` 번호와 **무관**.

### 같은 턴 필수 4종 (구현이 있을 때)

| # | 문서 |
|---|------|
| 1 | `29_vision_requests_report.md` §대화창 — **맨 위** 행 |
| 2 | `25_ai_work_log.md` — **맨 위** |
| 3 | `plans/YYMMDD-N.md` — `done` |
| 4 | `current_session_plan.md` |

→ **채팅 답변 전** `chat_request_recording.md` §6 자가 점검.

### `29` §대화창 — 표 열

| 열 | 내용 |
|----|------|
| ID | `C-YYMMDD-N` |
| 요청일 | `YYYY-MM-DD` |
| 요청 요약 | 사용자 말 한 줄 |
| 상태 | ✅ · 🔶 · 📖 · ⏸ |
| 계획·처리 | `plans/…` · 파일·H-xxx |

### vision.md와 구분

| 출처 | 기록 위치 | ID |
|------|-----------|-----|
| Cursor **채팅** 직접 지시 | `29` §대화창 | `C-YYMMDD-N` |
| **`vision.md` 추가요청** #N | `29` §상세(1~#59) · `27` | `#N` (C-ID **금지**) |
| **starter·`02`·감사** 점검만 | `25` · `plans` | 없음 |

- 대화 요청 → **`vision.md`에 쓰지 않음** · §대화창에만 `C-…`.
- vision 추가요청 완료 → §상세 `#N` **처리 방법** 갱신 (§대화창에 C-ID 넣지 않음).
- 분류 결정 트리 → [`chat_request_recording.md`](./chat_request_recording.md) §0.

→ 운영: `23_agent_operating_rules.md` §대화창

## AI 작업 일지 (`for_human/25_ai_work_log.md`)

- **매 세션 종료 시 필수** (`23` 체크리스트)
- **대화창 직접 요청**도 동일 (계획 파일 링크 필수)
- **문서 맨 위**에 새 항목 삽입 (맨 아래 추가 금지)
- 제목 줄: `### YYYY-MM-DD HH:mm:ss — 제목`
- 표에 **계획** · **한 일** · **당신이 할 일** 포함
- **계획**란: `plans/YYMMDD-N.md` 링크 + 해당 파일 `## 오늘 목표 (한 줄)` 요약 (경로만 적지 않음)

## for_human 문서 추가 규칙 (doit.md)

- 코딩 거의 모름 가정
- 쉬운 언어·비유·단계별 설명
- 예시 코드·동작 흐름·“왜” 설명
- 초보자 오해 포인트 명시
- **골라 읽기** 가능 + **전부 읽으면** 전체 이해
- API 문서 수준이 아닌 **교육 자료** 수준
- 코드 **의도**·**사고방식** 설명

→ 상세: `for_human/24_how_to_read_docs.md`

### 꼼꼼 관리 (에이전트)

- 기능 변경 시 **교차 참조 문서 표** (`23_agent_operating_rules.md` §문서 꼼꼼 관리)에 있는 파일을 한 세션에서 함께 맞출 것

- `for_human` 가이드는 **스크린 경로·Secret Name·완료 확인**까지 쓸 것 (추상적 한 줄만 쓰지 않음)
