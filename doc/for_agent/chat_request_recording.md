# 대화창 요청 — 기록 업무지침 (에이전트 정본)

> **인간용 표:** `for_human/29_vision_requests_report.md` §대화창에서 요청한 내용  
> **운영 규칙:** `23_agent_operating_rules.md` · **양식:** `26_document_standards.md` · **온보딩:** `starter.txt` §0·§대화창

`vision.md`가 **아닌**, 사용자가 **이 Cursor 대화창**에서 직접 시킨 모든 업무(기능·문서·H-xxx·질문 후 구현)는 아래를 **같은 턴**에 끝낸다.  
**코드만 반영하고 `29`·`25`를 나중에 쓰는 것 = 규칙 위반** (C-260522-11 사례).

---

## 1. 적용 범위

| 포함 | 제외 |
|------|------|
| 기능·버그·UI·Edge·SQL·doc 수정 요청 | `vision.md` 본문 수정 (금지) |
| 「H-003 완료」 반영 | 순수 잡담 (기록 생략) |
| 질문 → 답변 후 사용자가 구현 요청 | |
| doc/지침만 고치는 요청 (본 문서처럼) | |

---

## 2. ID 부여 (`C-YYMMDD-N`)

1. `for_human/29_vision_requests_report.md` §대화창 표 **맨 위~아래**에서 오늘 날짜 `C-YYMMDD-*` 중 **가장 큰 N**을 찾는다.
2. 새 요청 = **N+1** (예: 마지막이 `C-260522-16`이면 다음은 `C-260522-17`).
3. **한 사용자 메시지에 서로 다른 업무가 여러 개**면 업무마다 ID·plan 행을 나눈다.  
   **한 덩어리**(같은 기능의 계획+구현+doc)면 plan 1개 + C-ID 1개.

`plans/YYMMDD-N.md`의 N과 `C-YYMMDD-N`의 N은 **같을 필요 없음** (plan은 그날 파일 개수, C는 대화 요청 순번).

---

## 3. 필수 흐름 (한 요청당)

```
[착수 전]  plans/YYMMDD-N.md 새 파일 + current_session_plan 인덱스
           (「바로 해줘」면 채팅 승인 생략 가능)

[수행]     구현 · vision.md 수정 금지

[같은 턴 종료 — 아래 4개는 구현이 있으면 필수]
  ① 29 §대화창 표 맨 위에 C-YYMMDD-N 행
  ② 25_ai_work_log 맨 위 항목 (계획 링크 + 한 일 + 당신이 할 일)
  ③ plans/YYMMDD-N.md → done + 실행 결과
  ④ current_session_plan.md 인덱스

[선택]     02 · 05 · 17 · 10/11/14 등 (기능·테스트·인수인계 변경 시)
```

**질문만(📖):** `29`에 「📖」행 추가 또는 생략, `25`에 3줄 이하 (선택).

---

## 4. `29` 표 — 맨 위 행 템플릿

```markdown
| C-YYMMDD-N | YYYY-MM-DD | (사용자 요청 한 줄) | ✅ / 🔶 / 📖 / ⏸ | [`YYMMDD-N.md`](../for_agent/plans/…) · 변경 요약 |
```

| 상태 | 의미 |
|------|------|
| ✅ | 완료 |
| 🔶 | 진행 중 (다음 턴 이어감) |
| 📖 | 안내만, 코드 없음 |
| ⏸ | H-xxx 등 인간 대기 |

---

## 5. `25` 일지 — 맨 위 항목 템플릿

```markdown
### YYYY-MM-DD HH:mm:ss — (제목, C-ID 표기)

| 항목 | 내용 |
|------|------|
| **계획** | [`YYMMDD-N.md`](../for_agent/plans/YYMMDD-N.md) — (한 줄 목표) |
| **한 일** | (실제 변경 파일·Edge·SQL) |
| **당신이 할 일** | (인간 확인 1줄, 없으면 「없음」) |
```

---

## 6. 응답 보내기 전 자가 점검 (필수)

구현·doc 수정을 **이번 턴**에 했다면, 채팅 전송 전에 모두 체크:

- [ ] `29` §대화창 **맨 위**에 오늘 `C-YYMMDD-N` 행이 있는가?
- [ ] `25` **맨 위**에 같은 요청 일지가 있는가?
- [ ] `plans/YYMMDD-N.md`가 `done`이고 `current_session_plan`에 링크되는가?
- [ ] 사용자에게 **「기록 위치」**(`29` 표 ID)를 한 줄 안내했는가?

**하나라도 없으면** 답변을 보내지 말고 doc부터 채운다.
