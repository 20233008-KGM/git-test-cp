# 프로젝트 목적 및 철학

## 문제의식

기존 대학 수업의 팀 프로젝트에는 다음과 같은 구조적 문제가 존재한다.

### 1. 관계 형성의 어려움

- 팀원들끼리 서로 어떤 사람인지 잘 모른다.
- 교수 또한 학생들의 역량과 관심사를 충분히 알기 어렵다.
- 결과적으로 라포 형성과 협업 효율이 떨어진다.

### 2. 피드백 구조의 비효율성

- 교수와 팀 사이의 빠른 피드백 루프가 어렵다.
- 프로젝트 진행 상황을 실시간으로 공유하기 힘들다.
- 문제 해결 과정이 단절된다.

### 3. 팀플의 휘발성

- 수업 종료 후 팀플 결과물과 과정이 사라진다.
- 후배 세대가 과거 프로젝트의 작업물과 노하우를 참고할 수 없다.
- 개인 차원에서도 자신의 성장 기록이 축적되지 않는다.

### 4. 개인 성장 이력의 부재

- 어떤 문제를 해결했는지,
- 어떤 역할을 맡았는지,
- 어떤 기술을 사용했는지가
개인의 장기적인 이력으로 남지 않는다.

# 프로젝트 목표

이 프로젝트는 대학 팀플을 단순한 과제가 아니라:

- 사람과 사람을 연결하고,
- 협업 과정을 기록하며,
- 성장 데이터를 축적하는

**지속 가능한 협업 아카이브 플랫폼**으로 재구성하는 것을 목표로 한다.

# 핵심 기능

## 1. 수강생 네트워크

같은 수업을 듣는 학생들의 정보를 한 화면에서 탐색할 수 있다.

### 주요 기능

- 전체 수강생 프로필 조회
- 기술 스택 태그 등록
    - Python
    - JavaScript
    - React
    - 디자인 시스템
    - 기타 커스텀 태그
- 상세 프로필 조회
- 1대1 채팅 기능

### 목적

- 협업 상대 탐색
- 관심사 기반 연결
- 빠른 라포 형성
- 팀 구성 효율 향상

---

## 2. 팀플 워크플레이스

프로젝트 진행 과정을 기록하고 공유하는 협업 공간.

### 주요 기능

### 중간 작업 결과 업로드

- 같은 수업을 듣는 누구나 업로드 내용을 열람 가능
- 작업 히스토리 기록

### 트러블슈팅 시스템

- 현재 해결 중인 문제 등록
- 댓글 기반 인사이트 획득
- 문제 해결 과정 기록

### 목적

- 교수자의 학생 작업물 접근성을 높임
- 협업 과정 가시화
- 문제 해결 지식 축적
- "지금 무엇을 하고 있는지" 잊지 않게 도움
- 모든 팀간에 정보공유를 활성화하여 모든 학생의 빠른 성장을 도모

---

## 3. 마이페이지

종료된 프로젝트를 개인 성장 데이터로 정리하는 공간.

- 종료된 팀플 자동 수집
- AI 기반 프로젝트 요약
- a4 용지 비율로 정형화된 리포트 생성

### 목적

- 개인 성장 기록 축적
- 포트폴리오 자동화
- 협업 경험 아카이빙

# 프로젝트 철학

## 1. 팀플은 결과보다 과정이 중요하다

최종 발표 자료만 남는 것이 아니라,
어떤 고민과 문제 해결 과정을 거쳤는지가 기록되어야 한다.

---

## 2. 협업은 기억되어야 한다

대부분의 대학 팀플은 학기 종료와 동시에 사라진다.

이 프로젝트는:

- 작업물,
- 피드백,
- 시행착오,
- 문제 해결 과정

모두를 장기적으로 축적하는 것을 목표로 한다.

---

## 3. 사람 기반 협업 탐색

기존 LMS는 "수업 중심" 구조다.

이 프로젝트는:

- "사람"
- "관심사"
- "기술"
- "협업 스타일"

중심으로 연결을 만든다.

---

## 4. 학생의 성장 데이터를 남긴다

"무엇을 만들었는가" 뿐 아니라:

- 어떤 문제를 해결했는지
- 어떤 역할을 수행했는지
- 어떻게 성장했는지를

지속적으로 기록한다.
